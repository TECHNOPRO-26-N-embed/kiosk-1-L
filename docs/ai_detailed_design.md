## 1.2 各関数詳細設計
### replace_products
- **目的**: メインCSV（products.csv）と追加CSV（replace_products.csv）の指定商品番号で内容を入れ替える
- **入出力**:
  - 入力: メイン商品配列、追加商品配列、入れ替え対象商品番号リスト
  - 出力: 入れ替え後のメイン商品配列、追加商品配列
- **アルゴリズム**:
  1. 入れ替え対象商品番号ごとに、両CSVから該当商品を検索
  2. 両方に存在する場合、内容を入れ替える
  3. どちらかに存在しない場合はエラー記録
  4. 入れ替え後、products.csvの件数が50件を超えないことを確認
  5. 入れ替え操作の内容をログファイル（replace_log.csv等）に記録
  6. 結果を返す
- **エラー処理**:
  - 商品番号がどちらかに存在しない場合はエラー記録し、他の番号は処理継続
  - 上限超過時はエラー表示し処理中断

### authenticate_admin
- **目的**: 管理者パスワード認証（固定文字列照合）
- **入出力**:
  - 入力: 入力パスワード文字列
  - 出力: 認証結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. パスワード入力を受け取る
  2. 設定ファイル（ai_admin.conf等）から固定パスワード文字列を読み込む
  3. 入力値と設定値を比較
  5. 一致→1、不一致→0を返す
- **エラー処理**:
  - 入力が空文字の場合はエラー表示し再入力
  - 設定ファイルが読めない場合はエラー表示し、管理機能を利用不可とする

## 2. 設定ファイル設計

### 2.1 ファイル名例
- ai_admin.conf

### 2.2 ファイル内容例
```
# 管理者パスワード（平文）
admin_password=admin123
```

### 2.3 運用・セキュリティ
- パスワード変更時は設定ファイルの文字列を更新する
- 設定ファイルのパーミッションを制限し、一般利用者が閲覧できないようにする
- プログラムは起動時または認証時にこのファイルを読み込む

### calc_change
- **目的**: 釣銭計算
- **入出力**:
  - 入力: 支払金額、合計金額
  - 出力: 釣銭金額
- **アルゴリズム**:
  1. 支払金額 - 合計金額 を計算
  2. 結果を返す
- **エラー処理**:
  - 支払金額 < 合計金額の場合はエラー

### calc_total_price
- **目的**: 合計金額計算
- **入出力**:
  - 入力: 単価、個数
  - 出力: 合計金額
- **アルゴリズム**:
  1. 単価 × 個数 を計算
  2. 結果を返す
- **エラー処理**:
  - 単価または個数が1未満の場合はエラー

### input_integer
- **目的**: 整数入力（バリデーション付き）
- **入出力**:
  - 入力: プロンプト文字列、最小値、最大値
  - 出力: 入力値（整数）
- **アルゴリズム**:
  1. 入力値を文字列で受け取る
  2. 数値変換できるか判定
  3. 範囲内か判定
  4. OKなら値を返す、NGならエラー表示し再入力
- **エラー処理**:
  - 数値変換不可、範囲外はエラー

### load_products
- **目的**: 商品マスタCSV読込
- **入出力**:
  - 入力: ファイルパス
  - 出力: 商品配列
- **アルゴリズム**:
  1. CSVファイルを開く
  2. 1行ずつパースし構造体配列に格納
  3. 配列を返す
- **エラー処理**:
  - ファイルオープン失敗時はエラー
  - パース失敗時は該当行をスキップしエラー表示

### save_products
- **目的**: 商品マスタCSV書込
- **入出力**:
  - 入力: 商品配列、ファイルパス
  - 出力: 書込結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. ファイルを開く
  2. 配列内容を1行ずつCSV形式で出力
  3. 正常終了で1、失敗で0を返す
- **エラー処理**:
  - ファイル書込失敗時はエラー

### append_purchase_history
- **目的**: 購入履歴CSV追記
- **入出力**:
  - 入力: 購入情報構造体、ファイルパス
  - 出力: 書込結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. ファイルを追記モードで開く
  2. 購入情報をCSV形式で1行出力
  3. 正常終了で1、失敗で0を返す
- **エラー処理**:
  - ファイル書込失敗時はエラー

### show_main_menu
- **目的**: メインメニュー表示・選択
- **入出力**:
  - 入力: なし
  - 出力: メニュー番号（0:終了, 1:購入, 2:在庫管理）
- **アルゴリズム**:
  1. メニューを表示
  2. input_integerで番号入力
  3. 結果を返す
- **エラー処理**:
  - 範囲外番号はエラー

### show_product_list
**目的**: 商品一覧表示（ページング対応）
**入出力**:
  - 入力: 商品配列、ページ番号
  - 出力: なし
**アルゴリズム**:
  1. 指定ページ（例：1ページ10件）分の商品名・温度・価格・在庫を一覧表示
  2. 在庫0は売切表示
  3. ページ数が複数ある場合、「次へ」「前へ」などの選択肢を表示
  4. 利用者の入力でページ遷移（ページ番号を増減）
**エラー処理**:
  - 商品配列が空の場合はエラー
  - 範囲外ページ番号はエラー表示し再入力

### select_product
- **目的**: 商品選択処理
- **入出力**:
  - 入力: 商品配列
  - 出力: 選択商品インデックス
- **アルゴリズム**:
  1. 商品一覧を表示
  2. input_integerで商品番号入力
  3. 該当商品が在庫0ならエラー再入力
  4. OKならインデックス返す
- **エラー処理**:
  - 番号不正、在庫0はエラー

### input_quantity
- **目的**: 購入個数入力処理
- **入出力**:
  - 入力: 在庫数
  - 出力: 購入個数
- **アルゴリズム**:
  1. input_integerで1以上在庫以下を入力
  2. 結果を返す
- **エラー処理**:
  - 範囲外はエラー

### input_payment
- **目的**: 支払金額入力処理
- **入出力**:
  - 入力: 合計金額
  - 出力: 支払金額
- **アルゴリズム**:
  1. input_integerで合計金額以上を入力
  2. 結果を返す
- **エラー処理**:
  - 不足時はエラー

### show_purchase_result
- **目的**: 購入結果表示
- **入出力**:
  - 入力: 商品名、個数、合計金額、支払金額、釣銭
  - 出力: なし
- **アルゴリズム**:
  1. 購入内容・釣銭を表示
  2. 完了メッセージ表示
- **エラー処理**:
  - なし

### manage_stock
- **目的**: 在庫管理処理（補充）
- **入出力**:
  - 入力: 商品配列
  - 出力: 補充後商品配列
- **アルゴリズム**:
  1. 商品一覧を表示
  2. input_integerで商品番号入力
  3. input_integerで補充数入力
  4. 在庫上限超過はエラー
  5. 問題なければ在庫加算
- **エラー処理**:
  - 番号不正、上限超過はエラー

### error_message
- **目的**: エラーメッセージ表示
- **入出力**:
  - 入力: エラー内容文字列
  - 出力: なし
- **アルゴリズム**:
  1. エラー内容を表示
  2. 再入力を促す
- **エラー処理**:
  - なし
