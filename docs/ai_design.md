# 自動販売機システム 基本設計書

## 1. 文書情報
- 文書名: 基本設計書
- 対象システム: 自動販売機システム（飲料専用）
- 作成日: 2026-05-19
- 参照文書: 要件定義書

## 2. 目的と適用範囲
本書は、要件定義書で整理した要求事項を実装可能な設計情報に具体化することを目的とする。

適用範囲は以下とする。
- 飲料商品のみを販売対象とする
- 利用者は一般利用者（購入）と管理者（在庫変更）とする
- 画面は CUI（テキストベース）とする

## 3. システム概要
### 3.1 システム構成
- 実行方式: C言語によるコンソールアプリケーション
- 商品データ管理: CSV ファイル
- 購入履歴管理: CSV ファイルへの追記

### 3.2 業務フロー概要
1. メインメニューを表示する
2. 利用者が購入機能または在庫管理機能を選択する
3. 処理完了後はメインメニューへ戻る
4. 終了選択時にプログラムを終了する

## 4. 画面設計（CUI）
### 4.1 画面一覧
| 画面ID | 画面名 | 役割 |
| :---: | :--- | :--- |
| S01 | メインメニュー | 購入、在庫管理、終了の選択 |
| S02 | 商品一覧・商品選択 | 販売中商品の一覧表示と商品選択 |
| S03 | 個数入力 | 購入個数の入力 |
| S04 | 支払入力 | 支払金額の入力と不足判定 |
| S05 | 購入結果表示 | 商品排出結果、釣銭、完了表示 |
| S06 | 在庫管理 | 対象商品選択、補充数入力、更新結果表示 |

### 4.2 画面入力ルール
- メニュー番号、商品番号、個数、金額は整数入力のみ許可する
- 数字以外の入力時はエラーメッセージを表示して同一画面で再入力とする
- 範囲外入力時はエラーメッセージを表示して同一画面で再入力とする

## 5. 画面遷移設計
### 5.1 画面遷移
1. S01 メインメニュー
2. S01 で 購入 を選択した場合は S02 へ遷移
3. S02 で商品確定後、S03 へ遷移
4. S03 で個数確定後、S04 へ遷移
5. S04 で支払成立時、S05 へ遷移
6. S05 表示後、S02 へ戻る（継続購入）または S01 へ戻る
7. S01 で 在庫管理 を選択した場合は S06 へ遷移
   - S06 では管理者認証を行う
   - 認証成功時に在庫管理画面へ遷移
   - 認証失敗時はエラーメッセージを表示し、S01 へ戻る
8. S06 完了後は S01 へ戻る
9. S01 で 終了 を選択した場合はプログラム終了

### 5.2 遷移時エラー方針
- 不正入力時は前画面へ戻さず、原則として同一画面で再入力する
- 在庫不足、支払不足は当該処理画面で再入力する

## 6. 機能設計
### 6.1 F01 商品一覧表示
- 商品名、温度区分、価格を一覧表示する
- 在庫 0 の商品は売切表示を行う

### 6.2 F02 購入機能
1. 商品番号を入力し対象商品を決定する
2. 個数を入力する
3. 合計金額を計算する（単価 × 個数）
4. 支払金額を入力する
5. 支払不足時は再入力する
6. 支払成立時は釣銭を計算する（支払金額 - 合計金額）
7. 在庫を減算し、購入結果を表示する

### 6.3 F03 購入履歴保存
- 購入成立時に購入履歴 CSV へ 1 行追記する
- 追記項目は購入日時、商品名、単価、個数、支払金額、釣銭とする
- 書き込み失敗時はエラーメッセージを表示する

### 6.4 F04 在庫管理
- 管理者が対象商品と補充数を入力する
- 在庫上限（50）を超える補充はエラーとする
- 妥当な入力の場合は在庫を加算し CSV に反映する

### 6.5 F05 メニュー機能
- 起点画面として機能し、購入・在庫管理・終了へ分岐する
- 範囲外メニュー番号は再入力とする

### 6.6 管理者認証機能
- 管理者認証を在庫管理機能の前段階で実施する
- 認証方法:
  1. 管理者パスワードを入力させる
  2. 入力されたパスワードをSHA-256でハッシュ化する
  3. 設定ファイル（例: admin.conf）からハッシュ値を読み込む
  4. 入力値のハッシュと設定ファイルのハッシュを比較する
  5. 一致すれば認証成功、在庫管理画面へ遷移
  6. 不一致の場合はエラーメッセージを表示し、メインメニューへ戻る
- パスワードのハッシュ化には SHA-256 を使用する
- パスワードハッシュは「設定ファイル（例: admin.conf）」から読み込む

### 6.7 管理者機能
- 在庫管理画面で以下の操作を提供する:
  1. 商品の補充
  2. 商品情報の編集（将来拡張）
- 認証成功後のみ操作可能とする

## 7. データ設計
### 7.1 商品マスタ（products.csv）
| 項目名 | 説明 | 型 | 必須 | 制約 |
| :--- | :--- | :--- | :---: | :--- |
| product_id | 商品識別子 | 整数 | 必須 | 1 以上、重複不可 |
| product_name | 商品名 | 文字列 | 必須 | 空文字不可 |
| temperature | 温度区分 | 文字列 | 必須 | hot または cool |
| price | 単価 | 整数 | 必須 | 1 以上 |
| stock | 在庫数 | 整数 | 必須 | 0 以上 50 以下 |

### 7.2 購入履歴（purchase_history.csv）
| 項目名 | 説明 | 型 | 必須 | 制約 |
| :--- | :--- | :--- | :---: | :--- |
| purchased_at | 購入日時 | 文字列 | 必須 | yyyy-mm-dd hh:mm:ss 形式 |
| product_name | 商品名 | 文字列 | 必須 | 空文字不可 |
| unit_price | 単価 | 整数 | 必須 | 1 以上 |
| quantity | 購入個数 | 整数 | 必須 | 1 以上 |
| total_price | 合計金額 | 整数 | 必須 | unit_price × quantity |
| paid_amount | 支払金額 | 整数 | 必須 | total_price 以上 |
| change_amount | 釣銭 | 整数 | 必須 | paid_amount - total_price |

### 7.3 データ件数制約
- 商品件数は最大 50 件
- 1 商品の在庫は最大 50 個

## 8. 入出力設計
### 8.1 主な入力
| 入力項目 | 入力元 | 形式 | 備考 |
| :--- | :--- | :--- | :--- |
| メニュー番号 | 利用者 | 整数 | 1:購入、2:在庫管理、0:終了 |
| 商品番号 | 利用者 | 整数 | 一覧表示中の有効番号 |
| 購入個数 | 利用者 | 整数 | 1 以上かつ在庫以下 |
| 支払金額 | 利用者 | 整数 | 合計金額以上 |
| 補充数 | 管理者 | 整数 | 1 以上かつ上限内 |

### 8.2 主な出力
- 商品一覧
- 合計金額
- 釣銭
- 購入完了メッセージ
- 在庫更新完了メッセージ
- エラーメッセージ

## 9. バリデーション設計
| 対象 | チェック内容 | エラー時対応 |
| :--- | :--- | :--- |
| 数値入力全般 | 数値以外を拒否 | エラー表示して再入力 |
| メニュー番号 | 定義範囲内か | エラー表示して再入力 |
| 商品番号 | 存在する番号か | エラー表示して再入力 |
| 購入個数 | 1 以上かつ在庫以下か | エラー表示して再入力 |
| 支払金額 | 合計金額以上か | エラー表示して再入力 |
| 補充後在庫 | 50 以下か | エラー表示して再入力 |

## 10. エラー処理設計
### 10.1 必須異常系
- 数字以外の入力
- 範囲外のメニュー番号、商品番号
- 在庫不足
- 支払金額不足
- CSV ファイルオープン失敗
- CSV 書き込み失敗

### 10.2 エラー時の共通方針
- プログラムを異常終了させない
- 利用者に原因が分かるメッセージを表示する
- 同一画面で再入力させるか、メインメニューに復帰する

## 11. CSV 更新設計
### 11.1 更新対象
- 在庫更新時: products.csv
- 購入成立時: products.csv と purchase_history.csv

### 11.2 更新順序（購入）
1. 入力値検証
2. 在庫減算後の値を計算
3. products.csv を更新
4. purchase_history.csv に追記
5. 成功メッセージを表示

### 11.3 失敗時方針
- ファイル処理失敗時はエラー表示し、処理継続可能な状態でメニューへ復帰する
- 更新失敗時の整合性確保は、将来拡張として再試行または一時ファイル方式を検討する

## 12. 非機能設計
### 12.1 可用性
- 不正入力でも停止しない
- エラー復帰経路を各画面に持つ

### 12.2 保守性
- 商品情報、在庫情報、購入履歴を CSV で分離管理する
- 入力処理、購入処理、在庫処理、ファイル処理を機能単位で分離する

### 12.3 拡張性
- 季節ごとの商品入替に対応できるよう、商品情報をデータ駆動で読み込む
- 商品項目追加に備えて CSV パーサを汎用化可能な構造にする

## 13. テスト観点（基本設計レベル）
- 正常系: 購入成立、在庫補充成立、終了
- 入力異常系: 非数値、範囲外番号
- 業務異常系: 在庫不足、支払不足、在庫上限超過
- ファイル異常系: 読込不可、書込不可
- 境界値: 在庫 0、在庫 50、商品件数 50

## 14. 前提・制約・未決事項
### 14.1 前提・制約
- 内部の硬貨、紙幣枚数は管理しない
- 商品数上限 50、在庫上限 50 を遵守する

### 14.2 未決事項
  → 導入する
  → メニューに戻す
  → purchase_history.csv を data/ フォルダに保存

## 15.2 各関数詳細設計

### authenticate_admin
- **目的**: 管理者パスワード認証（SHA-256ハッシュ照合）
- **入出力**:
  - 入力: 入力パスワード文字列
  - 出力: 認証結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. パスワード入力を受け取る
  2. SHA-256でハッシュ化
  3. 設定ファイル（admin.conf等）からハッシュ値を読み込む
  4. 入力値のハッシュと比較
  5. 一致→1、不一致→0を返す
- **エラー処理**:
  - 入力が空文字の場合はエラー表示し再入力
  - 設定ファイルが読めない場合はエラー表示し、管理機能を利用不可とする
# 16. 設定ファイル設計

## 16.1 ファイル名例
- admin.conf

## 16.2 ファイル内容例
```
# 管理者パスワードSHA-256ハッシュ
admin_password_hash=5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8
```

## 16.3 運用・セキュリティ
- パスワード変更時はハッシュ値を再生成し、設定ファイルを書き換える
- 設定ファイルのパーミッションを制限し、一般利用者が閲覧できないようにする
- プログラムは起動時または認証時にこのファイルを読み込む

### calc_change
- **目的**: 釣銭計算
- **入出力**:
  - 入力: 支払金額、合計金額
  - 出力: 釣銭金額
- **アルゴリズム**:
  1. 支払金額 - 合計金額 を計算
  2. 結果を返す
- **エラー処理**:
  - 支払金額 < 合計金額の場合はエラー

### calc_total_price
- **目的**: 合計金額計算
- **入出力**:
  - 入力: 単価、個数
  - 出力: 合計金額
- **アルゴリズム**:
  1. 単価 × 個数 を計算
  2. 結果を返す
- **エラー処理**:
  - 単価または個数が1未満の場合はエラー

### input_integer
- **目的**: 整数入力（バリデーション付き）
- **入出力**:
  - 入力: プロンプト文字列、最小値、最大値
  - 出力: 入力値（整数）
- **アルゴリズム**:
  1. 入力値を文字列で受け取る
  2. 数値変換できるか判定
  3. 範囲内か判定
  4. OKなら値を返す、NGならエラー表示し再入力
- **エラー処理**:
  - 数値変換不可、範囲外はエラー

### load_products
- **目的**: 商品マスタCSV読込
- **入出力**:
  - 入力: ファイルパス
  - 出力: 商品配列
- **アルゴリズム**:
  1. CSVファイルを開く
  2. 1行ずつパースし構造体配列に格納
  3. 配列を返す
- **エラー処理**:
  - ファイルオープン失敗時はエラー
  - パース失敗時は該当行をスキップしエラー表示

### save_products
- **目的**: 商品マスタCSV書込
- **入出力**:
  - 入力: 商品配列、ファイルパス
  - 出力: 書込結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. ファイルを開く
  2. 配列内容を1行ずつCSV形式で出力
  3. 正常終了で1、失敗で0を返す
- **エラー処理**:
  - ファイル書込失敗時はエラー

### append_purchase_history
- **目的**: 購入履歴CSV追記
- **入出力**:
  - 入力: 購入情報構造体、ファイルパス
  - 出力: 書込結果（成功:1, 失敗:0）
- **アルゴリズム**:
  1. ファイルを追記モードで開く
  2. 購入情報をCSV形式で1行出力
  3. 正常終了で1、失敗で0を返す
- **エラー処理**:
  - ファイル書込失敗時はエラー

### show_main_menu
- **目的**: メインメニュー表示・選択
- **入出力**:
  - 入力: なし
  - 出力: メニュー番号（0:終了, 1:購入, 2:在庫管理）
- **アルゴリズム**:
  1. メニューを表示
  2. input_integerで番号入力
  3. 結果を返す
- **エラー処理**:
  - 範囲外番号はエラー

### show_product_list
- **目的**: 商品一覧表示
- **入出力**:
  - 入力: 商品配列
  - 出力: なし
- **アルゴリズム**:
  1. 商品名・温度・価格・在庫を一覧表示
  2. 在庫0は売切表示
- **エラー処理**:
  - 商品配列が空の場合はエラー

### select_product
- **目的**: 商品選択処理
- **入出力**:
  - 入力: 商品配列
  - 出力: 選択商品インデックス
- **アルゴリズム**:
  1. 商品一覧を表示
  2. input_integerで商品番号入力
  3. 該当商品が在庫0ならエラー再入力
  4. OKならインデックス返す
- **エラー処理**:
  - 番号不正、在庫0はエラー

### input_quantity
- **目的**: 購入個数入力処理
- **入出力**:
  - 入力: 在庫数
  - 出力: 購入個数
- **アルゴリズム**:
  1. input_integerで1以上在庫以下を入力
  2. 結果を返す
- **エラー処理**:
  - 範囲外はエラー

### input_payment
- **目的**: 支払金額入力処理
- **入出力**:
  - 入力: 合計金額
  - 出力: 支払金額
- **アルゴリズム**:
  1. input_integerで合計金額以上を入力
  2. 結果を返す
- **エラー処理**:
  - 不足時はエラー

### show_purchase_result
- **目的**: 購入結果表示
- **入出力**:
  - 入力: 商品名、個数、合計金額、支払金額、釣銭
  - 出力: なし
- **アルゴリズム**:
  1. 購入内容・釣銭を表示
  2. 完了メッセージ表示
- **エラー処理**:
  - なし

### manage_stock
- **目的**: 在庫管理処理（補充）
- **入出力**:
  - 入力: 商品配列
  - 出力: 補充後商品配列
- **アルゴリズム**:
  1. 商品一覧を表示
  2. input_integerで商品番号入力
  3. input_integerで補充数入力
  4. 在庫上限超過はエラー
  5. 問題なければ在庫加算
- **エラー処理**:
  - 番号不正、上限超過はエラー

### error_message
- **目的**: エラーメッセージ表示
- **入出力**:
  - 入力: エラー内容文字列
  - 出力: なし
- **アルゴリズム**:
  1. エラー内容を表示
  2. 再入力を促す
- **エラー処理**:
  - なし
