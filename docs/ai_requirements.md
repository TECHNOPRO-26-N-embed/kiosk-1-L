# 自動販売機システム 要件定義書

## 1. 文書情報
- 文書名: 要件定義書
- 対象システム: 自動販売機システム（飲料専用）
- 目的: 顧客ヒアリング内容を基に、基本設計・詳細設計の入力となる要求事項を明確化する。

## 2. 背景・目的
本システムは、飲料の販売を行う自動販売機を想定したC言語アプリケーションである。
利用者による商品購入と、管理者による在庫管理を行えることを目的とする。

## 3. 適用範囲（スコープ）
- 対象: 飲料商品のみ（飲料以外は対象外）
- 想定利用者:
  - 一般利用者（購入操作）
  - 管理者（在庫変更操作）

## 4. 前提条件・制約
- 同時に扱う商品数は最大50種類とする。
- 1商品あたりの在庫上限は50個を目安とする。
- 自動販売機内部の硬貨・紙幣の枚数管理は行わない。
- 季節ごとに販売商品を入れ替える運用を想定する。
- 想定外入力（文字入力、不正な数値、存在しない選択肢など）があっても、プログラムは停止しないこと。

## 5. 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 | 商品一覧表示 | 販売中の飲料について、商品名・温度区分（hot/cool）・価格・在庫数を一覧表示する。 |
| F02 | 購入機能 | 商品選択、購入個数入力、支払金額計算、入金受付、釣銭計算、商品排出までの購入処理を行う。 |
| F03 | 購入履歴保存 | 購入日時、商品名、単価、購入個数、支払金額、釣銭などをCSVに追記保存する。 |
| F04 | 在庫管理 | 商品の在庫増減（購入による減算、補充による加算）を行い、CSVへ反映する。 |
| F05 | メニュー機能 | ユーザーが購入機能・在庫管理機能・終了操作へ遷移できる起点メニューを提供する。 |

## 6. 非機能要件（基本設計の前提）
- 可用性:
  - 入力異常時はエラーメッセージを表示し、再入力または前画面へ復帰する。
  - 異常入力でプログラムを異常終了させない。
- 保守性:
  - 商品情報・在庫情報・購入履歴はCSVで管理し、データ更新手順を明確化する。
- 拡張性:
  - 季節ごとの商品入替が可能なデータ構造・読み込み方式とする。

## 7. 基本設計への引き継ぎ事項
- 画面遷移は「メインメニュー」を起点に、購入系と在庫管理系に分岐する。
- データ設計では、最低限以下を管理対象とする。
  - 商品名
  - 温度
  - 価格
  - 在庫数
  - 購入日時
  - 購入個数
  - 支払金額
  - 釣銭
- エラー処理設計では、以下を必須シナリオとして扱う。
  - 数字以外の入力
  - 範囲外の商品番号・メニュー番号
  - 在庫不足
  - 支払金額不足
