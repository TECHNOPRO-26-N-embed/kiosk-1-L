## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* 販売する商品は飲料を想定する。飲料の種類・価格はチーム側で適当に決めていい。
* 飲料以外の商品は、今回は扱わない。
* 各商品の最大在庫数は、50個程度を目安とする。
* 想定外の入力や操作があっても、プログラムが止まらないようにする。
* 自動販売機の中の効果の枚数は考慮しない
* 同時に扱う商品は50種類
* 在庫は1種類あたり50個
* 季節ごとの製品入換あり

### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 |商品一覧表示 |商品の金額と商品名の一覧を表示する |
| F02 |購入機能 |商品選択　→　個数を入力　→　支払い金額を表示　→　購入継続か受け付ける　→　お金を受け取る　→　指定された商品を出す　→　お釣りの出力 |
| F03 |購入履歴保存 |csvファイルに変更を適用 |
| F04 |在庫管理 |入力をもとにcsvファイルを変更 |

## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
- 1.メインメニューを表示
1.商品購入機能もしくは管理者機能に飛ぶ

- 2.商品一覧表示機能
1.データを格納用の配列を用意
2.csvファイルを読み取る
3.csvファイルの商品名、金額を配列に保存
4.c配列の内容を1つずつ（商品名、金額、番号はプラスして）表示

- 3. 商品購入機能
1.	（お客様）商品を選択 (商品idを確保)
2.	（お客様）個数を入力（個数と在庫を比べ、個数を確保）
3.	（自販機）金額を表示（商品idに対する単位と個数を*）
4.	（お客様）お金を投入（投入金額と合計金額を比べ）
   4.1．（足りる）お釣りを計算、5に進む
   4.2．（足りない）エラーメッセージ表示、2に戻る
5.（自販機）商品とお釣りを出す（商品在庫数更新、日付記録）

- 4. 購入履歴保存
1. 情報を受け取る(商品名,商品の単価,合計金額,支払金額,おつり)
2. 購入日時を取得
3. 購入履歴を保存するcsvファイルを開く
4. csvファイルに、購入情報を1行追加
5. 書き込みが完了したら、csvファイルを閉じる
6. 保存に成功したら保存完了メッセージ
7. ファイルを開けない、書き込み失敗はエラーメッセージ

- 5.在庫管理
1. 在庫を変更する商品の情報を受け取る(商品名,変更する個数)
2. 商品情報が保存されているcsvファイルを開く
3. 対象の商品と一致するまで探す
4. 見つかったら在庫数を変更(購入の場合、在庫数から購入個数を引く。補充の場合、在庫数に補充個数を足す)
5. 保存
6. ファイルを閉じる
7. 成功したら完了メッセージ
8. 商品が見つからない、在庫数が不足しているときは、エラーメッセージ

- 6.支払関連機能
1.お客がいくら支払うかの入力を受ける
（この際、入力が0円or文字or足りない場合、それぞれのエラー文を出す
例：「金額が足りません」「数字を入力してください。」）
2.指定された商品を画面に文字で表示する
（例：商品名：コーラ　本数：n本）
3.入力された支払金額と合計金額に差がある場合、お釣りを出す
（例：お釣り：無し　ご購入ありがとうございました。キー入力で商品選択に戻ります。）
```

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 数字以外の文字が入力された場合 | "数字を入力してください"を出力して元の機能に戻る|
| 金額が不足している場合 | "金額が足りません"を出力して元の機能に戻る|
| 存在しないメニューを選んだ時 |"実在する商品番号を選んでください。"を出力し元の機能に戻る |
